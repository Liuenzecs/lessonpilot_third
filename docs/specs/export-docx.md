# Word 导出规格

## 目标

导出文件必须让老师可以提交、打印或继续小幅修改。

当前 MVP 以 `.docx` 为唯一正式导出格式。

## 数据来源

导出读取：

- `Document.content`
- `Task` 元信息

导出服务：

- `apps/api/app/services/export_service.py`

## 教案导出

包含：

- 标题
- 学科、年级、课时、课型
- 教学目标
- 教学重难点
- 教学准备
- 教学过程
- 板书设计
- 教学反思

规则：

- 教学过程使用表格。
- 家教场景可简化列。
- 教学反思即使为空，也保留填写区域。
- 默认只导出已确认 section，反思空白区除外。

## 学案导出

包含：

- 标题
- 学科、年级、班级、姓名、日期
- 学习目标
- 重点难点预测
- 知识链接
- 学习流程
- 达标测评
- 拓展延伸
- 自主反思

规则：

- 班级、姓名、日期可留空但要有位置。
- 学习流程中的自主学习、合作探究、展示提升来自 `learning_process`。

## 未来模板导入方向

学校模板应被解析成：

- 字段映射。
- section 顺序。
- 表格布局。
- 必须保留的空白区域。
- 静态学校文案。

不要把学校模板硬编码成全局唯一格式。

## 学校模板化导出

模板接口：

- `POST /api/v1/templates/school/preview`
- `POST /api/v1/templates/school/confirm`
- `GET /api/v1/templates/school/personal`
- `DELETE /api/v1/templates/school/{template_id}`

导出接口：

- `GET /api/v1/documents/{document_id}/export?format=docx&template_id=...`

规则：

- 模板归属于当前用户，不能被其他用户读取或用于导出。
- 模板保存为结构化 export spec，不保存真实学校样例进仓库。
- 模板导出优先使用模板的栏目顺序、元信息字段、教学过程表格列和空白区域。
- 模板不可访问或不存在时返回 404；不传模板时使用默认导出。
- v1 不承诺像素级复刻原 Word。

## 导出前体检

导出前体检接口：

- `POST /api/v1/documents/{document_id}/quality-check`

返回：

- `readiness`：`ready / needs_fixes / blocked`
- `summary`：老师可读的总体判断
- `issues`：需要处理的问题
- `warnings`：提交前提醒
- `suggestions`：可选优化
- `alignment_map`：目标、过程、评价对应关系

### 教案检查

- 标题、学科、年级是否存在。
- 教学目标、教学重难点、教学过程是否有内容且已确认。
- 教学过程是否至少 3 个环节。
- 每个教学过程环节是否包含环节名称、时长、教师活动、学生活动。
- 公立校和机构场景是否缺少设计意图。
- 总时长是否明显偏离课时。
- 教学准备、板书设计是否为空。
- 教学反思为空时只提醒，不阻断；导出仍保留填写区。

### 阻断语义

- `blocked`：文档几乎没有可导出的内容，或缺少标题、学科、年级等关键元信息。
- `needs_fixes`：可以继续导出，但正式提交前建议处理，例如核心 section 仍是 `pending`。
- `ready`：主要导出风险已通过。

前端可以在 `needs_fixes` 时提示老师先处理，也允许老师确认后继续导出；`blocked` 时暂不建议继续导出。

### 质量检查 2.0

在原有导出风险之外，继续检查：

- 教学目标是否具体、可观察、可评价。
- 教学过程是否承接教学目标。
- 教学重难点是否在过程中展开。
- 学生活动是否只是“认真听讲”等被动表述。
- 学案达标测评是否覆盖学习目标。
