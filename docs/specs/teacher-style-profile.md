# 老师个人风格档案规格

## 目标

记录当前用户可解释、可编辑的备课表达偏好，并在生成 / 重写时作为 prompt 约束注入。

## 数据归属

- 风格档案归属于当前用户。
- 只用于当前用户的生成和重写。
- 不进入公共知识库，不跨用户共享。

## 字段

- `enabled`：是否启用风格记忆。
- `objective_style`：教学目标常用写法。
- `process_style`：教学过程组织偏好。
- `school_wording`：学校提交常用措辞。
- `activity_preferences`：常用课堂活动。
- `avoid_phrases`：尽量避免的套话或表达。
- `sample_count`：后续自动学习可记录样本数量，v1 默认 0。

## API

- `GET /api/v1/style-profile`
  - 获取当前用户风格档案；如果不存在，返回默认空档案。
- `PUT /api/v1/style-profile`
  - 更新当前用户风格档案。

## Prompt 注入规则

- 只有 `enabled=true` 且至少一个风格字段非空时注入。
- 注入内容只作为表达风格参考。
- 仍需遵守 section schema、质量规则、RAG 引用和学校模板约束。
- 禁用后不注入任何个人风格内容。

## 降级语义

- 档案不存在：按普通生成处理。
- 档案为空：按普通生成处理。
- 风格服务异常：跳过风格提示，不影响生成主链路。
