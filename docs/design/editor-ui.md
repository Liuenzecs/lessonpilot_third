#编辑器页面

> **⚠️ 已过时（2026-05-05）**：本文件为早期 Notion 风格的编辑器设计。当前视觉权威以根目录 `DESIGN.md`（教师文档桌设计系统）为准。本文档仅保留作为设计历史参考。

## 设计哲学

一句话：像 Notion 一样简洁，像 Word 一样熟悉，像教案本一样有结构感。

老师的真实使用场景：晚上 10 点备明天的课，或者课间 10 分钟改一下练习题。他们需要的是"打开就能用，3 秒内知道该点哪里"。所以整个产品的 UX 基调是：零学习成本、极少的页面层级、所见即所得。

---

## 核心页面结构

整个产品只需要 4 个主要视图，不要更多：

```
┌─────────────────────────────────────────────────────┐
│  ① 首页/工作台    ② 创建向导    ③ 编辑器    ④ 设置  │
└─────────────────────────────────────────────────────┘
```

### ① 首页 / 工作台

这是老师每次打开产品看到的第一个页面。不要做成 dashboard 堆满数据图表，老师不需要那些。

```
┌──────────────────────────────────────────────┐
│  LessonPilot                        头像 ▾  │
├──────────────────────────────────────────────┤
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │  + 开始备课                          │    │
│  │    选择学科和主题，AI 帮你快速生成    │    │
│  └──────────────────────────────────────┘    │
│                                              │
│  最近备课                          搜索 🔍   │
│  ┌────────┐ ┌────────┐ ┌────────┐           │
│  │ 一元二次│ │ 光合作用│ │ 古诗词 │           │
│  │ 方程    │ │        │ │ 鉴赏   │           │
│  │ 数学·八年│ │ 生物·七年│ │ 语文·九年│        │
│  │ 3天前   │ │ 1周前   │ │ 2周前  │           │
│  └────────┘ └────────┘ └────────┘           │
│                                              │
└──────────────────────────────────────────────┘
```

设计要点：
- "开始备课"按钮要大、要醒目，是页面的视觉焦点
- 最近备课用卡片展示，每张卡片显示：标题、学科·年级、时间
- 卡片点击直接进入编辑器，不要多一层"详情页"
- 不要侧边栏导航。页面少，顶部导航或者直接在首页完成一切入口就够了

### ② 创建向导

不要做成一个复杂的长表单。用分步向导，每步只问一个问题，降低认知负担：

```
步骤 1/3
┌──────────────────────────────────────┐
│                                      │
│   选择学科                           │
│                                      │
│   ┌──────┐ ┌──────┐ ┌──────┐       │
│   │ 语文  │ │ 数学  │ │ 英语  │       │
│   └──────┘ └──────┘ └──────┘       │
│   ┌──────┐ ┌──────┐ ┌──────┐       │
│   │ 物理  │ │ 化学  │ │ 生物  │       │
│   └──────┘ └──────┘ └──────┘       │
│   ┌──────┐ ┌──────┐                 │
│   │ 历史  │ │ 更多… │                 │
│   └──────┘ └──────┘                 │
│                                      │
│                          下一步 →    │
└──────────────────────────────────────┘

步骤 2/3：选择年级（同样的卡片选择）
步骤 3/3：输入课题主题（一个输入框 + 可选的补充说明）
```

设计要点：
- 每步一屏，大按钮/卡片点选，不要下拉框
- 最后一步可以有一个可选的文本框让老师补充具体要求（比如"重点讲解配方法"）
- 点击"生成教案"后直接跳转到编辑器页面，在编辑器里展示生成进度
- 整个向导 3 步，10 秒内完成

### ③ 编辑器（核心，80% 的时间花在这里）

这是产品的灵魂页面，需要最多的设计思考。

```
┌──────────────────────────────────────────────────────────┐
│  ← 返回    一元二次方程 · 数学 · 八年级        导出 ▾   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌─ 大纲导航 ─┐  ┌─ 编辑区域 ──────────────────────┐   │
│  │             │  │                                  │   │
│  │ ● 教学目标  │  │  教学目标                        │   │
│  │ ● 教学重难点│  │  ─────────                       │   │
│  │ ● 导入环节  │  │  1. 理解一元二次方程的概念        │   │
│  │ ● 新授环节  │  │  2. 掌握配方法解方程              │   │
│  │ ● 练习巩固  │  │  3. ...                          │   │
│  │ ● 课堂小结  │  │                                  │   │
│  │             │  │  教学重难点                       │   │
│  │             │  │  ─────────                       │   │
│  │             │  │  重点：配方法的基本步骤           │   │
│  │             │  │  难点：...                        │   │
│  │             │  │                                  │   │
│  │             │  │  ┌─ AI 待确认 ──────────────┐    │   │
│  │             │  │  │ ✨ 导入环节              │    │   │
│  │             │  │  │ 通过生活实例引入...       │    │   │
│  │             │  │  │                          │    │   │
│  │             │  │  │    ✓ 接受    ✗ 拒绝    ↻ 重新生成│   │
│  │             │  │  └──────────────────────────┘    │   │
│  │             │  │                                  │   │
│  └─────────────┘  └──────────────────────────────────┘   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

编辑器布局的关键决策：

左侧大纲导航（可折叠）：
- 教案天然有结构（教学目标、重难点、教学过程、练习、小结），左侧用大纲树展示
- 点击可快速跳转到对应 section
- 在小屏幕上自动折叠，不占空间
- 这个导航让老师一眼看到教案的完整骨架，强化"结构感"

中间编辑区域（主体）：
- 占页面 70-80% 的宽度
- 每个 block 悬停时显示左侧的拖拽手柄 + 右侧的操作菜单（⋮）
- 操作菜单包含：AI 重写、删除、上移/下移、转换类型
- 编辑体验尽量接近 Word/Notion：点击即编辑，回车新建段落，Tab 缩进

不要右侧 AI 聊天面板：
- 很多产品喜欢在右边放一个 AI 对话框，但对这个产品来说是错误的。老师不需要和 AI 聊天，他们需要的是"点哪改哪"
- AI 交互应该是内联的（inline），不是分离的

### Block 的交互设计

每个 block 的交互模式：

```
普通状态：
┌──────────────────────────────────┐
│  教学目标                        │
│  1. 理解一元二次方程的概念        │
│  2. 掌握配方法解方程              │
└──────────────────────────────────┘

悬停状态：
⠿ ┌──────────────────────────────┐ ⋮
   │  教学目标                     │
   │  1. 理解一元二次方程的概念     │
   │  2. 掌握配方法解方程           │
   └──────────────────────────────┘

选中文本后的浮动工具栏：
         ┌─────────────────────────────┐
         │ B  I  U  │ ✨AI润色  ✨AI扩写 │
         └─────────────────────────────┘
   │  1. 理解一元二次方程的概念     │
   │  2. 掌握 ███配方法███ 解方程   │  ← 选中的文本

AI 生成中的 block：
┌─ ✨ 正在生成 ─────────────────────┐
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  │
│  正在生成"导入环节"...            │
└──────────────────────────────────┘

AI 生成完成待确认：
┌─ ✨ AI 生成 ─────────────────────┐
│  导入环节（5分钟）                │  ← 浅紫色/浅蓝色背景
│  通过展示生活中的抛物线现象...     │     区分于正常内容
│                                  │
│  ✓ 接受    ✗ 拒绝    ↻ 重新生成  │
└──────────────────────────────────┘
```

这个"待确认"状态是 UX 的关键。用一个明显但不刺眼的背景色（浅紫色或浅蓝色）标记 AI 生成的内容，底部三个操作按钮。老师接受后，背景色消失，变成普通 block。

### 编辑器底部的快捷操作栏

```
┌──────────────────────────────────────────────┐
│  + 添加段落  │  ✨ AI 补充内容  │  📝 添加练习  │
└──────────────────────────────────────────────┘
```

在编辑器最底部放一个简洁的操作栏，让老师可以快速追加内容。"AI 补充内容"点击后弹出一个小输入框，老师写一句话描述需求，AI 生成新的 block 追加到末尾。

### 导出交互

不要做成一个独立页面。在编辑器顶部的"导出"按钮点击后，弹出一个下拉面板：

```
┌─────────────────────┐
│  导出为              │
│  ┌───────────────┐  │
│  │ 📄 Word 文档   │  │
│  └───────────────┘  │
│  ┌───────────────┐  │
│  │ 📋 PDF 文档    │  │  ← MVP 可灰显，标注"即将支持"
│  └───────────────┘  │
│                     │
│  预览导出效果 →      │
└─────────────────────┘
```

"预览导出效果"打开一个模态框，展示接近最终 Word 排版的预览。这个预览不需要 100% 精确，但要让老师有信心"导出来的东西是能用的"。

## AI 生成过程的 UX

这是最容易做砸的地方。老师点击"生成教案"后，不能让他们对着一个空白页面干等。

推荐的体验流程：

```
第 1 秒：跳转到编辑器页面，显示教案的结构骨架（灰色占位）
┌──────────────────────────────────┐
│  ✨ 正在为你生成教案...           │
│                                  │
│  ┌─ 教学目标 ──────────────┐     │
│  │  ░░░░░░░░░░░░░░░░░░░░  │     │  ← 骨架屏/shimmer
│  └─────────────────────────┘     │
│  ┌─ 教学重难点 ────────────┐     │
│  │  ░░░░░░░░░░░░░░░░░░░░  │     │
│  └─────────────────────────┘     │
│  ┌─ 教学过程 ──────────────┐     │
│  │  ░░░░░░░░░░░░░░░░░░░░  │     │
│  └─────────────────────────┘     │
└──────────────────────────────────┘

第 3-5 秒：第一个 section 生成完毕，实际内容替换骨架
┌──────────────────────────────────┐
│  ✨ 正在生成... (2/6)             │
│                                  │
│  ┌─ 教学目标 ──────────────┐     │
│  │  1. 理解一元二次方程...  │     │  ← 已生成，可以开始看了
│  │  2. 掌握配方法...        │     │
│  └─────────────────────────┘     │
│  ┌─ 教学重难点 ────────────┐     │
│  │  ░░░░░░░░░░░░░░░░░░░░  │     │  ← 还在生成
│  └─────────────────────────┘     │
└──────────────────────────────────┘

第 10-15 秒：全部生成完毕
┌──────────────────────────────────┐
│  ✅ 教案已生成，你可以开始编辑    │  ← 顶部提示条，几秒后自动消失
│  ...完整的教案内容...             │
└──────────────────────────────────┘
```

这个"骨架 → 逐段填充"的体验比"转圈等待 10 秒然后一次性出现"好太多。老师在等待的过程中已经可以开始阅读前面生成好的部分了。

## 视觉设计方向

色彩：
- 主色调建议用偏暖的蓝色或靛蓝色（专业、可信赖、不冷冰冰）
- AI 相关的元素用浅紫色或浅蓝色标记（和正常内容区分）
- 背景用纯白或极浅的暖灰（长时间编辑不伤眼）
- 不要用鲜艳的红绿黄做主色，教育产品不需要"活泼感"，需要"安心感"

字体：
- 编辑区域用系统默认的衬线或无衬线字体，16px 起步
- 不要用花哨的字体。老师看的是内容，不是设计
- 行高 1.6-1.8，段间距要充足。教案内容密度高，需要呼吸感

整体气质：
- 参考 Notion 的简洁感 + 飞书文档的结构感
- 不要参考那些花里胡哨的 AI 产品（渐变背景、粒子动画、深色主题）
- 老师打开这个产品的感觉应该是"一个干净的工作台"，不是"一个科技产品"

---

## 容易踩的 UX 坑

1. 不要把 AI 做成聊天界面。老师不想"对话"，他们想"点一下就出结果"。所有 AI 交互都应该是：触发 → 等待 → 确认/拒绝，三步完成。

2. 不要隐藏导出按钮。导出是老师使用这个产品的终极目的，它应该始终可见在编辑器顶部，不要藏在菜单里。

3. 不要在编辑器里堆太多工具栏按钮。Notion 的做法是对的——默认干净，悬停/选中时才出现操作。老师需要的操作其实很少：改文字、调顺序、AI 重写、删除。

4. 不要让保存状态不可见。编辑器顶部要有一个小小的保存状态指示："已保存" / "保存中..." / "未保存的更改"。老师最怕的就是"我改了半天结果没存上"。

5. 不要在生成过程中锁死整个页面。AI 生成教案时，已经生成好的部分应该可以开始编辑。不要用一个全屏 loading 遮罩挡住一切。

6. 不要忽略空状态设计。新用户第一次进来，工作台是空的。这时候不要显示一个空白页面，而是显示一个友好的引导："👋 欢迎！点击下方按钮开始你的第一次备课"，配一个大大的 CTA 按钮。

7. 移动端先不做复杂适配，但编辑器页面至少要在平板上能用。很多老师会用 iPad 备课。

---

总结：这个产品的 UI/UX 核心就是"结构可见、AI 内联、操作极简"。老师打开就看到清晰的教案骨架，AI 生成的内容以待确认状态内联展示，所有操作都在编辑器内完成不跳页面。视觉上追求"安静的工作台"感觉，不追求科技感。
