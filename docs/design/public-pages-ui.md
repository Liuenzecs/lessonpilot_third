#公域页面

## 先解决命名问题

核心功能区的名字，我建议叫"备课台"。

理由：
- "工作台"太通用，像项目管理工具
- "编辑器"太技术，老师听不懂
- "教案区"太窄，后续还有练习、题目
- "备课台"直接对应老师的核心动作，一听就懂，而且有"工作台面"的隐喻——打开就是你的备课桌

产品内所有入口按钮统一叫"进入备课台"或"开始备课"。

## 全局导航栏（公域）

所有公域页面共享同一个顶部导航栏：

```
┌──────────────────────────────────────────────────────────────┐
│  🎯 LessonPilot    功能介绍    定价    帮助中心    登录  开始使用 │
└──────────────────────────────────────────────────────────────┘
```

设计要点：
- Logo + 产品名在最左侧，点击回首页
- 中间是页面链接，最多 3-4 个，不要堆太多
- 右侧"登录"是文字链接，"开始使用"是实心按钮（主 CTA）
- 导航栏固定在顶部（sticky），滚动时加一层微妙的阴影
- 移动端折叠为汉堡菜单
- 背景白色或透明（在首页 hero 区域可以透明，滚动后变白）

登录后导航栏变为：

```
┌──────────────────────────────────────────────────────────────┐
│  🎯 LessonPilot    帮助中心                    备课台  头像 ▾ │
└──────────────────────────────────────────────────────────────┘
```

"备课台"按钮替代"开始使用"，头像下拉菜单包含：账户设置、退出登录。

## 响应式策略

MVP 阶段不需要做完美的移动端适配，但要确保以下几点：

公域页面（首页、定价、FAQ 等）：必须完全响应式。这些页面老师可能在手机上看到（比如朋友分享了链接）。用 Tailwind 的响应式工具类就能搞定。

备课台列表页：平板和手机上卡片变成 2 列或 1 列，可用。

编辑器页面：平板上可用（左侧大纲折叠），手机上显示一个提示"请在电脑或平板上使用编辑器以获得最佳体验"。不要强行在手机上做编辑器适配，体验一定很差。

## 页面间的跳转流程总结

```
首次访问用户：
Landing 首页 → 点击"免费开始备课" → 注册页 → 注册成功 → 备课台（空状态）
→ 点击"开始第一次备课" → 创建向导 → 编辑器（AI 生成中）→ 编辑 → 导出

老用户：
Landing 首页 → 点击"登录" → 登录页 → 备课台
→ 点击已有教案卡片 → 编辑器 → 继续编辑 → 导出
→ 或点击"开始备课" → 创建向导 → 编辑器

从搜索引擎/分享链接进入：
任意公域页面 → 导航栏"开始使用" → 注册/登录 → 备课台
```

---

整体就是这些页面了。核心原则贯穿始终：页面层级尽量少（大部分操作 2 次点击内完成），视觉干净不花哨，文案说人话不说官话，每个页面都有一个明确的"下一步该做什么"的引导。
