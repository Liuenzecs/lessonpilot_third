# 当前任务：Phase 11 真实样本验收与个人资料复用闭环（待验收）

## 本阶段目标

让老师上传过的个人旧资料在下一次备课中真的被用起来：

- 同主题备课时推荐当前账号的旧教案、讲义或 PPT 大纲
- 生成时可选择参考“我的资料库”，并显示命中/未命中状态
- 生成结果留下 section 级“我的资料”引用
- 导出前体检中的高频问题可一键转成待确认修订
- 使用脱敏合成样本覆盖模板、旧教案、PPT 大纲、复用生成和质量修复验收

## 已授权范围

- 仍聚焦语文 MVP，不扩展全学科
- 个人资料默认私有，不进入公共知识包，不跨用户共享
- v1 使用关键词、标题、课题和正文片段匹配，不强依赖 embedding
- 质量修复只生成待确认修订，不替老师自动验收
- 阶段结束停在验收口，不自动提交 / 推送，不进入下一阶段

## 已实现

- `GET /api/v1/personal-assets/recommend` 可按学科、年级、课题和关键词推荐当前用户私有资料片段。
- 生成请求支持 `use_personal_assets` 与 `personal_asset_ids`，并拒绝跨用户资产。
- 生成流新增 `asset_status`，前端展示“我的资料”命中、未命中、未启用或降级状态。
- 个人资料引用写入 `section_references`，前端 tooltip 区分“知识库资料”和“我的资料”。
- 创建页和编辑器都可开启“参考我的资料库”，编辑器可按当前资料重新整理。
- 导出前体检面板新增“按建议调整”，支持目标具体化、过程对齐、重难点入过程、学生活动可执行化等待确认修订。

## 验收标准

- 老师上传旧 `.docx` 或 `.pptx` 后，下一次同主题备课能看到个人资料推荐
- 开启“参考我的资料库”后，生成过程能显示个人资料命中/未命中状态
- 至少 2 个生成 section 能留下“我的资料”引用，并在前端与公共知识库引用区分展示
- 其他用户无法推荐、检索、引用当前用户的个人资料
- 至少 3 类质量检查问题能一键生成待确认修订，且不自动覆盖 confirmed 状态为已确认
- 学校模板、旧教案导入、上课包、默认 Word 导出、公共 RAG 引用链路不退化
- 前后端测试、类型检查、构建与静态检查通过

## 本轮待验收

- [ ] 上传一份与课题相关的旧 `.docx` 或 `.pptx` 到个人资料库，确认保存后只在当前账号可见。
- [ ] 在创建页输入同主题课题，开启“整理初稿时参考我的资料库”，确认能看到推荐资料。
