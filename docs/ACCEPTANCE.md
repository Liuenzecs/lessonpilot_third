# 手动验收脚本

## 当前主链路验收

目标：确认 LessonPilot 从创建到导出已经稳定可演示。

### 1. 登录与备课台

- 注册或登录一个测试账号。
- 进入备课台。
- 确认空状态、任务卡片、搜索、复制、删除入口显示正常。

通过标准：
- 页面无异常跳转。
- 未登录时会回到登录页。
- 已登录时能看到备课台。

### 2. 创建备课

- 点击“开始备课”。
- 选择语文、年级、课题。
- 分别测试教案、学案、教案+学案。
- 填写补充说明。

通过标准：
- 能创建任务并进入编辑器。
- 新任务会自动创建对应文档。

### 3. Section 级生成

- 新建任务后等待自动生成。
- 观察每一节生成过程。

通过标准：
- 事件按 `section_start / section_delta / section_document / section_done / document_done` 进行。
- 每个 section 完成后内容持续可见。
- 不出现“生成后短暂消失”。

### 4. 编辑与保存

- 修改一个文本 section。
- 修改一个列表 section。
- 修改教学过程或题目 section。
- 等待自动保存。
- 刷新页面。

通过标准：
- 修改不会丢失。
- 保存状态能回到已保存。
- 学案 `self_study / collaboration / presentation` 均保存到 `learning_process`。

### 5. 局部改写

- 对一个已有 section 点击调整。
- 测试改写、补充展开、压缩表达。

通过标准：
- 改写前会保存本地改动。
- 改写完成后 section 内容更新。
- 文档版本增加。

### 6. RAG 引用

- 使用已入库知识能命中的课题，例如：
  - `红楼梦薛宝钗人物分析`
  - `春 朱自清 第一课时`
  - `桃花源记文言文教学`
- 生成后查看 section 顶部参考资料。
- 另外测试一个未覆盖课题，例如 `论语十二章`。

通过标准：
- 生成时能看到知识增强状态。
- 命中知识包时状态显示 domain 和资料数量。
- 未命中时提示本次按普通生成处理。
- 命中 RAG 时 `section_references` 非空。
- 前端显示参考资料徽标和 tooltip。
- `RAG_ENABLED=false` 时能优雅退化为普通生成。

### 6.1 RAG 诊断接口

- 登录后调用 `POST /api/v1/knowledge/diagnose`。
- 分别传入上述命中课题和未命中课题。

通过标准：
- 命中课题返回 `ready` 或 `matched_empty`。
- 未命中课题返回 `unmatched`。
- 返回结果不暴露 embedding 或数据库内部异常。

### 7. Word 导出

- 确认若干 section。
- 点击“导出前体检”。
- 查看体检结果中的阻断项、提醒项和可选优化。
- 在目标、教学过程或测评题中保留一段公式，例如 `\\(S=\\frac{1}{2}ah\\)` 或 `\\(x^2=9\\Rightarrow x=\\pm3\\)`。
- 点击导出 Word。
- 打开 Word 文件。

通过标准：
- 空文档或缺少关键元信息时显示 `blocked`。
- 核心 section 仍为待确认时显示 `needs_fixes`，并提示“当前 Word 导出不会包含这部分内容”。
- 内容完整且已确认时显示 `ready`。
- 文件能打开。
- 标题、元信息、教学目标、过程表格等格式正常。
- 仅导出已确认内容，反思保留空白区。
- 公式不是 LaTeX 源码文本，而是 Word 中可选中、可编辑的原生公式对象。

### 8. 旧 Word 教案导入

- 进入备课台，点击“导入旧教案”。
- 上传一份 `.docx` 教案，内容包含教学目标、教学重难点、教学过程、板书设计、教学反思。
- 查看结构化预览、warnings 和未识别内容。
- 修正标题、年级、课题等基本信息。
- 点击“确认导入并进入编辑器”。

通过标准：
- 至少能识别教学目标、教学重难点、教学过程、板书设计、教学反思中的 4 类。
- 未识别内容出现在预览中，不会静默丢弃。
- 进入编辑器后导入内容均为待确认状态。
- 老师可以逐节确认、编辑、改写和导出。
- 非 `.docx` 文件会被拒绝，超过 5MB 的文件会提示拆分后再导入。

### 9. 学校 Word 模板

- 进入备课台，点击“学校模板”。
- 上传一份 `.docx` 学校教案模板，模板中包含元信息、教学目标、教学过程表格、教学反思或签字区。
- 查看字段映射、栏目顺序、表格列和空白区域。
- 保存为个人模板。

通过标准：
- 能看到结构化模板预览。
- 能保存为当前账号的个人模板。
- 其他账号看不到该模板。
- 删除模板后列表会刷新。

### 10. 模板化 Word 导出

- 打开一份内容完整且已确认的教案。
- 在编辑器中选择刚保存的学校模板。
- 点击导出 Word。
- 打开 Word 文件检查。

通过标准：
- 文件能打开。
- 栏目顺序按模板规格组织。
- 教学过程表格列使用模板识别出的列。
- 教学反思、签字、审核等空白区域被保留。
- 不选择模板时仍能使用默认 Word 导出。

### 11. 教学质量检查 2.0

- 准备一份目标空泛、重难点未进过程、学生活动偏被动的教案。
- 点击“导出前体检”。
- 再准备一份目标、过程、评价较完整的教案进行对照。

通过标准：
- 体检能指出目标空泛、过程未承接目标、重难点未展开、学生活动偏被动等问题。
- 结果包含目标、过程、评价的对应关系。
- `blocked / needs_fixes / ready` 语义仍然清晰。

### 12. 上课包与个人资料库

- 在已确认教案中点击“生成上课包”。
- 查看学案草稿、PPT 大纲和课堂口播稿摘要。
- 进入“个人资料库”，分别上传一份 `.docx` 和 `.pptx`。
- 保存资料，再用另一个账号查看资料库。

通过标准：
- 上课包三类内容都生成，且默认待确认。
- `.docx` / `.pptx` 能预览识别内容、复用建议和 warning。
- 保存后的资料只出现在当前账号。
- 个人资料删除后不会继续出现在列表中。

### 13. 个人资料复用与质量修复

- 在“个人资料库”上传一份与课题相关的旧 `.docx` 或 `.pptx`。
- 回到创建页，输入同一课题，开启“整理初稿时参考我的资料库”。
- 进入编辑器并等待生成完成，观察生成状态和 section 顶部参考资料。
- 准备一份目标空泛、学生活动偏被动或重难点未进入过程的教案，运行导出前体检。
- 点击体检问题中的“按建议调整”。

通过标准：
- 创建页或编辑器能看到同主题个人资料推荐。
- 生成时能看到“我的资料”命中、未命中或降级状态。
- 命中个人资料后，至少 2 个 section 能看到“我的资料”引用，且与公共知识库引用区分显示。
- 另一个账号不能看到、推荐或引用当前账号的个人资料。
- 质量修复会生成待确认内容，对应 section 状态保持 pending。
- 默认 Word 导出、学校模板导出、公共 RAG 引用和上课包生成不退化。

### 14. Phase 12 教师文档桌 UI

- 打开公域首页，检查第一屏是否能直接看到备课文档、目录、质量检查、引用和导出能力。
- 登录后进入备课台，查看顶部工具区和最近备课队列。
- 点击“开始备课”，检查创建页是否以课题输入为主，并展示模板、个人资料和知识增强提示。
- 打开一份已有教案，检查编辑器是否为顶部工具栏、左侧目录、中间纸张画布、右侧检查栏。
- 分别打开“导入旧教案”“学校模板”“个人资料库”，检查上传、预览、确认、列表和删除入口是否仍然清晰。

通过标准：
- 首页第一屏能说明 LessonPilot 是“备课文档 + 质量检查 + Word 导出”。
- 备课台能看出每份备课的下一步动作。
- 创建页不再像长表单，课题输入和资料匹配是主流程。
- 编辑器像在线文档，section 状态、引用、质量检查、学校模板、个人资料和上课包均可见。
- 页面无明显文本溢出、控件重叠、卡片套卡片或过度装饰。
- 现有创建、生成、section 确认、AI 改写、RAG 引用、个人资料引用、质量修复、上课包、学校模板、Word 导出不退化。

### 15. 数学公式生成与展示

- 创建一份含数学内容的教案或学案，例如“勾股定理 第一课时”或“二次函数配方法练习”。
- 等待生成完成，检查 section 正文、测评题、答案和解析中的公式。
- 手动编辑一段公式文本并保存。
- 导出 Word 后打开文件。

通过标准：
- 生成 / 重写内容不出现 `\f`、制表符等 JSON 控制字符乱码。
- 编辑器中 `\\(...\\)`、`$$...$$` 公式能显示为正常数学排版。
- 老师仍能编辑原始文本，预览随文本更新。
- Word 导出中的常见公式成为可编辑的原生公式对象。

### 16. Phase 13 首份可交付试用

- 使用一个新账号或清空任务的测试账号进入备课台。
- 点击“用样例体验”。
- 在创建页检查是否自动填入 `朱自清《春》 第一课时`、语文、七年级、公立校教案等字段。
- 不修改字段，点击“进入文档桌”。
- 等待生成完成后运行导出前体检并导出 Word。
- 按 `docs/fixtures/phase-13-real-trial-samples.md` 继续演示旧教案导入、学校模板、个人资料库和公式导出。

通过标准：
- 空备课台能清楚给出样例体验、从课题开始、导入旧教案三条路径。
- 样例预填后老师仍可编辑字段和补充说明。
- 样例创建仍使用原有创建接口，不新增后端分支。
- 生成、质量检查、Word 导出链路能接上。
- 既有旧教案导入、学校模板、个人资料库入口仍然明显。

## 验收记录规则

验收通过后再更新 `docs/NEXT.md` 为“等待下一步指示”或下一个经用户确认的 Cycle。

不要在未验收前自动进入全学科、真实 PPT 导出、PDF/OCR、学校后台或部署阶段。
