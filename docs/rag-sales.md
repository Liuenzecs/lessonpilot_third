# RAG 对外介绍话术

## 一句话

LessonPilot 会在生成语文备课内容前，先自动参考你已经整理好的教研资料，再把这些资料变成更稳、更快可编辑的教案和学案草稿。

## 面向客户的核心表达

- 这不是一个只会“瞎编”的生成器，而是会先参考已有资料的备课助手。
- 对文学类和重点篇目课题，系统会自动检索已入库的教研材料，再生成教学目标、教学环节和练习题。
- 老师不仅能拿到结果，还能看到每一段内容参考了哪份资料、哪一回、哪条知识卡。
- 老师始终保留最终决定权，引用只是“参考依据”，不是强制替代老师判断。

## 推荐讲法

### 讲效率

“以前老师整理好的资料很难在生成时真正用起来，现在系统会先把相关资料找出来，再按课题逐节生成到编辑器里，省掉反复翻资料和手工拼接的时间。”

### 讲质量

“不是让模型空想，而是让模型先看你沉淀下来的教研资料，再生成目标、流程和题目，所以产出的内容更贴近真实备课。”

### 讲可控

“老师能看到参考出处，知道这段内容参考了什么资料，再决定要不要采用。”

## 适合演示的场景

- 《红楼梦》整本书阅读
- 重点人物分析
- 经典情节赏析
- 诗词与文学手法讲解
- 带有教研沉淀的专题复习课

## 演示顺序建议

1. 先展示已有知识包
2. 创建一个文学类课题
3. 展示 section 级逐节生成
4. 展示 section 顶部的“参考资料”徽标
5. 打开 tooltip 说明来源
6. 强调老师可以继续编辑、确认、导出

## 不建议的表述

- 不要说“这是问答机器人”
- 不要说“系统一定正确”
- 不要说“有了它老师就不需要备课”

更合适的说法是：

- “它把老师已有资料变成更高效的生成底座”
- “它提高首稿质量，但最终判断权仍在老师”
