# LessonPilot 产品重新规划 v2

> 状态：已确认（2026-04-14 起实施，2026-05-05 更新）
> 日期：2026-04-14（2026-05-05 更新公益定位）
> 背景：Codex 实现的 Phase 0-7 虽然按清单完成，但核心产品体验不达标——AI 生成质量差、编辑器交互差、导出质量差、缺少老师视角。需要从产品层面重新规划。
> 
> **2026-05-05 更新**：原规划中 Phase 3 的"完整计费 / 会员 / 接支付"已取消。项目现定位为纯公益免费，不做商业化。详见 `docs/CURRENT_PROGRESS_AUDIT.md`。

---

## 一、产品定位

**一句话**：AI 备课助手 — 一份备课，教案学案一起出。

**核心价值**：新老师输入学科/年级/课题 → AI 流式生成教案和/或学案 → 改两下 → 导出学校能接受的 Word 文档 → 15 分钟完成备课。

**核心产品原则**（不变，但执行要到位）：
1. **结构感优先** — 给骨架，AI 填肉，老师调整
2. **AI 是引擎不是主角** — 不是聊天，是在结构化框架内填充
3. **老师掌控感** — AI 生成内容以"待确认"状态呈现
4. **老师视角** — 从三种老师的实际工作流出发设计

---

## 二、目标用户

三种用户群体，设计时需要兼顾：

| 用户 | 核心场景 | 教案需求 | 学案需求 |
|------|---------|---------|---------|
| ① 公立校新手老师（0-3 年） | 学校要求提交教案，应对检查 | 学校标准格式（严格） | 正式导学案 |
| ② 大学生家教老师 | 快速备课，内容好用就行 | 简化版（注重内容） | 讲义/练习 |
| ③ 机构新手老师/小机构老师 | 每天多节课，追求效率+质量 | 中等规范 | 讲义为主 |

**设计策略**：创建备课时提供「使用场景」选项（公立校 / 家教 / 培训机构），影响模板选择和导出格式。

---

## 三、MVP 学科：语文

选择理由：
- 语文教案结构最复杂（三维目标、朗读指导、写作手法分析），做完语文其他学科更容易
- 语文是所有学科的基础，用户基数最大
- 语文教案的 AI 生成最具挑战性，能充分验证 prompt 质量

## 四、核心用户流

### 4.1 创建备课（一站式）

```
┌─────────────────────────────────────────────────┐
│  开始备课                                         │
│                                                   │
│  学科：语文 ✓        年级：[七年级 ▾]              │
│  课题：[从百草园到三味书屋             ]           │
│  课时：[第 1 课时 ▾]   课型：[新授课 ▾]            │
│                                                   │
│  生成内容：                                        │
│    ○ 只生成教案    ○ 只生成学案    ○ 教案 + 学案    │
│                                                   │
│  使用场景：                                        │
│    ○ 公立校    ○ 家教    ○ 培训机构                │
│                                                   │
│  补充要求（可选）：                                 │
│  ┌─────────────────────────────────────────────┐ │
│  │ 需要重点讲鲁迅的童年视角...                    │ │
│  └─────────────────────────────────────────────┘ │
│                                                   │
│               [ 开始生成 → ]                       │
└─────────────────────────────────────────────────┘
```

### 4.2 AI 流式生成

- **真正的 token-by-token 流式输出**，像 ChatGPT 那样文字逐字出现
- 教案和学案分 tab 显示，可以切换查看
- 生成过程中可以随时停止
- 已生成的部分可以立即编辑
- 进度指示：先出教学目标和重难点（老师最关心），再出教学过程

### 4.3 文档式编辑

- 界面像一份可折叠的 Word 文档，不是 block 编辑器
- 每个 section（教学目标、教学过程等）是一个可展开/折叠的区域
- 点击任意位置直接编辑，不需要选择 block 类型
- AI 操作以 section 为粒度：重写、扩写、精简
- 简单富文本：加粗、列表、缩进
- 自动保存

### 4.4 导出

- Word 导出按学校标准格式
- 教案：表头 + 三维目标 + 表格式教学过程 + 板书设计
- 学案：留有学生书写空间 + 题号和答题线
- 可选择同时导出教案和学案

## 五、内容结构

### 5.1 教案结构（语文）

```
教案 (LessonPlan)
├── 基本信息
│   ├── 课题名称
│   ├── 学科 / 年级 / 课时 / 课型
│   └── 使用场景（公立校/家教/培训机构）
│
├── 教学目标
│   ├── 知识与技能：掌握重点字词、理解文章内容...
│   ├── 过程与方法：通过朗读和小组讨论，提高...
│   └── 情感态度与价值观：体会作者对童年生活的...
│
├── 教学重难点
│   ├── 重点：理解文章两部分的对比关系
│   └── 难点：体会鲁迅语言的深层含义
│
├── 教学准备
│   └── 多媒体课件、课文朗读音频
│
├── 教学过程（主体，占 80%）
│   ├── 一、导入新课（5 分钟）
│   │   ├── 教师活动：播放鲁迅故居视频，提问...
│   │   ├── 学生活动：观看视频，思考并回答...
│   │   └── 设计意图：激发兴趣，建立对鲁迅的...
│   ├── 二、初读课文，整体感知（10 分钟）
│   │   └── （同上结构）
│   ├── 三、精读品析，深入理解（20 分钟）
│   │   └── （同上结构）
│   ├── 四、总结提升（5 分钟）
│   │   └── （同上结构）
│   └── 五、作业布置（5 分钟）
│       └── （同上结构）
│
├── 板书设计
│   └── 结构化的板书布局
│
└── 教学反思（留空，课后填写）
```

**使用场景对结构的影响**：

| Section | 公立校 | 家教 | 培训机构 |
|---------|-------|------|---------|
| 教学目标 | 三维目标（严格） | 简化学习目标 | 核心素养目标 |
| 教学重难点 | 必须 | 可选 | 必须 |
| 教学准备 | 详细 | 简化 | 中等 |
| 教学过程 | 表格式（教师/学生/意图） | 简化流程 | 中等 |
| 板书设计 | 必须 | 不需要 | 可选 |
| 教学反思 | 留空 | 不包含 | 可选留空 |

### 5.2 学案结构（语文）

```
学案 (StudyGuide)
├── 基本信息
│   ├── 课题名称 / 学科 / 年级 / 课时
│   └── 班级 / 姓名 / 日期（学生填写）
│
├── 学习目标（用"我能..."口吻）
│   ├── 我能掌握本课重点字词
│   ├── 我能有感情地朗读课文
│   └── 我能理解作者的童年情感
│
├── 重点难点预测
│   └── 学习时可能遇到的障碍
│
├── 知识链接
│   └── 前置知识：鲁迅简介、写作背景
│
├── 学习流程
│   ├── 自主学习（A/B 级）
│   │   ├── A 级（识记）：给加点字注音、解释词语
│   │   └── B 级（理解）：概括文章大意、划分段落
│   ├── 合作探究（B/C 级）
│   │   ├── 分析"百草园"部分的描写手法
│   │   └── 比较百草园和三味书屋的不同
│   └── 展示提升（C 级）
│       └── 小组代表展示讨论成果
│
├── 达标测评（当堂检测）
│   ├── 基础题（A/B 级）
│   └── 提高题（C 级）
│
├── 拓展延伸（D 级选做）
│   └── 仿写一段自己童年趣事
│
└── 自主反思
    ├── 知识盘点：本节课学到了什么
    └── 心得感悟：还有哪些疑问
```

**使用场景对学案的影响**：

| Section | 公立校 | 家教 | 培训机构 |
|---------|-------|------|---------|
| 学习目标 | 三维目标转写 | 简化 | 核心能力 |
| 知识链接 | 详细 | 简化 | 中等 |
| 自主学习 | A/B 分层 | B 级为主 | A/B 分层 |
| 合作探究 | 小组讨论题 | 个人思考题 | 小组讨论题 |
| 达标测评 | 完整测试 | 重点练习 | 完整测试 |
| 拓展延伸 | D 级选做 | 不含 | D 级选做 |

## 六、AI 策略

### 6.1 模型

- **初期**：DeepSeek + MiniMax 双模型
- **架构**：抽象 LLM Provider 接口，支持随时切换/增加
- **未来**：按成本/质量动态路由（初期不做）

### 6.2 Prompt 设计原则

**当前问题**：Prompt 把 JSON block 结构当作输出格式，AI 花大量 token 输出结构标记而非教学内容。

**新策略**：
1. **角色设定**："你是一位有 20 年教学经验的语文特级教师，擅长教案和学案设计"
2. **分层 prompt**：系统 prompt（角色+学科知识+课程标准） + 结构 prompt（输出 JSON Schema）
3. **教学内容质量优先**：prompt 重点放在"教什么、怎么教、为什么这样教"，而非 JSON 格式
4. **学科模板嵌入**：不同学科使用不同的教学过程模板
5. **场景适配**：不同使用场景影响 prompt 中的格式和详细程度要求

### 6.3 流式输出

- 真正的 token-by-token 流式（非 SSE 状态事件）
- 生成顺序：基本信息 → 教学目标 → 重难点 → 教学准备 → 教学过程（按环节）→ 板书设计
- 如果勾选了学案，教案完成后自动接续学案
- 老师随时可以停止生成

## 七、UI/UX 方向

### 7.1 视觉风格：中式现代风

**核心理念**：融入中国传统美学元素，以现代表达呈现。让产品有文化属性和辨识度。

**配色**：

| 角色 | 色值 | 用途 |
|------|------|------|
| 宣纸白 | `#faf8f3` | 主背景 |
| 象牙色 | `#f0ebe0` | 卡片/次级背景 |
| 墨色 | `#2c2c2c` | 主文字 |
| 石青 | `#3a7ca5` | 主强调色（按钮、链接） |
| 胆黄 | `#d4a843` | 辅助强调色（点缀、标签） |
| 朱砂 | `#c75450` | 警示色 |
| 竹青 | `#5a8f6c` | 成功色 |
| 灰墨 | `#8a8a8a` | 次要文字 |

**字体**：
- 标题：思源宋体（Noto Serif SC）或霞鹜文楷（LXGW WenKai）— 有书卷气
- 正文：系统无衬线体（PingFang SC / Microsoft YaHei）
- 代码/标注：等宽字体

**卡片/容器**：
- 8-12px 圆角（不是 18-28px 的大圆角）
- 细线边框（1px，低透明度）
- 轻投影或无投影
- 传统纹样作为微妙的装饰元素（如角落的水墨线条）

**整体感觉**：
- 大量留白，让内容呼吸
- 标题用宋体/楷体营造文化感
- 强调色用石青而非通用蓝色
- 导出文档的排版也体现中式审美

### 7.2 关键页面设计方向

**Landing 页**：
- Hero 区：水墨风渐变背景 + 大号宋体标题
- 产品预览：展示编辑器和导出效果
- 痛点/功能：用传统色彩的现代卡片
- 整体感觉：不像 SaaS，像一本精心排版的书籍

**编辑器**：
- 背景：宣纸色
- 编辑区像一张白纸，周围是柔和的阴影
- Section 折叠条用细线分隔
- AI 生成中：文字逐渐出现，像毛笔在书写
- 侧边大纲：简洁的文字列表，不用花哨的样式

**导出的 Word 文档**：
- 表头有传统装饰线条
- 标题用宋体
- 教学过程用表格排版
- 整体像一份正式的教学文档

### 7.3 交互改进（四个重点）

1. **流式生成体验**：文字逐字出现，进度条显示当前生成到哪个 section
2. **文档式编辑**：像编辑 Word，直接点击文字，不用理解 block
3. **创建一站式**：所有选项在一页完成，不跳转不翻页
4. **导出质量**：Word 文档排版专业，符合学校标准

## 八、MVP 范围

### 第一阶段：核心重做（MVP）

**必须做好**：
1. 用户注册/登录（可复用现有）
2. 创建备课（一站式：学科/年级/课题/课时/课型 + 教案/学案选择 + 使用场景 + 可选要求）
3. AI 流式生成教案和/或学案
4. 文档式编辑器（Section 级折叠/编辑）
5. Section 级 AI 重写

- PDF 导出
- 历史版本
- 管理后台、分析管道、SSR、Sentry
- 复杂计费系统（保留免费/专业两档的基础架构即可）
- 暗色模式
- Onboarding、反馈 widget
- 多学科支持（只做语文）

### 第二阶段：体验增强

- 学案 ABCD 四级分层
- 更多 AI 操作（精简、扩写、选段润色）
- PDF 导出
- 数学/英语学科模板
- 教案模板库

### 第三阶段：运营基础

- 完整计费系统
- 管理后台
- 分析管道
- SEO/SSR
- Sentry
- 更多学科

## 九、技术架构方向（高层）

### 保留
- 前端：Vue 3 + TypeScript + Vite + Pinia + TanStack Query
- 后端：FastAPI + Python + SQLModel + PostgreSQL
- 项目结构：pnpm workspace monorepo
- 用户认证：JWT（可复用现有）
- Tiptap 编辑器（但使用方式需改变）

### 重做
- **内容模型**：从 8 种 block 类型 → 教案结构化模板 + 学案结构化模板
- **AI 通信**：从 SSE 状态事件 → 真正的 token-by-token 流式
- **AI Prompt**：从技术化的 block 生成 → 教学内容质量优先的结构化生成
- **编辑器**：从 block editor → section-based document editor
- **导出**：从通用文档 → 学校标准教案格式
- **UI**：从通用 SaaS 蓝 → 中式现代风
- **LLM Provider**：支持 DeepSeek + MiniMax，可扩展

### 新增
- 使用场景（公立校/家教/培训机构）影响模板和 prompt
- 教案/学案平级选择
- 语文学科专项模板

## 十、与现有代码的关系

**可以复用**：
- 用户认证（注册/登录/JWT）— 已验证可用
- 项目结构（monorepo、pnpm workspace）
- 基础设施（Docker Compose、Alembic 迁移）
- HTTP client、路由守卫等基础工具

**需要重写**：
- 内容模型（block → 教案/学案结构）

- 编辑器（block editor → document editor）
- 导出服务（通用 → 学校标准格式）
- 全部 UI（视觉重设计）
- 创建页面（三步向导 → 一站式）

**需要删除/暂时移除**：
- 管理后台（admin）
- 分析管道（analytics）
- SSR 相关代码
- Sentry 集成
- 计费系统的大部分逻辑
- Onboarding、反馈 widget
