# Phase 16: 迁移竞争力提升 — 让老师"回不去"

> 基于 2026-04-29 竞品迁移分析制定。目标：从 ClassIn / 科大讯飞 / 希沃迁移来的老师，使用后形成不可逆依赖。

## 背景分析

LessonPilot 的结构化备课 + 文档桌体验 + 风格记忆是正确的差异化方向。但当前产品卡在"导出 Word"这个终点——老师导出后工作流就断了。

竞品的护城河：
- **希沃**：课件（EN5 白板）+ 课件库共享 → 上课必须用
- **科大讯飞**：学校集采 + 海量题库 + 个性化学习 → 换平台成本高
- **ClassIn**：备课→上课→作业→沟通一体化 → 不用切换工具

LessonPilot 的破局点：不是做"大而全"，而是做"备课这件事上最好用的工具"，让老师从"备课→上课→反思→再备课"形成完整闭环。

## 三阶段计划

### 第一阶段：降低迁移门槛（让老师愿意试）

目标：扫除老师从其他平台迁移过来的最大障碍。

| 优先级 | 功能 | 理由 | 状态 |
|--------|------|------|------|
| **P0** | 课件大纲生成 + PPTX 导出 | 连接"备课"和"上课"，解决"备完课还要去希沃重做课件"的核心矛盾 | [x] |
| **P0** | 语文重点篇目分层题库 | 学案题目不再完全依赖 AI 即时生成，有精选题目支撑 | [x] |
| **P1** | 旧教案批量导入（从文件夹） | 降低迁移的数据搬运成本 | [ ] |
| **P1** | 教研组分享链接（只读/可评论） | 使单人工具能进入学校场景，让老师可以推荐给同事 | [ ] |

### 第二阶段：建立使用习惯（让老师每天用）

目标：从"偶尔生成"变成"每周依赖"，覆盖老师的完整备课周期。

| 优先级 | 功能 | 理由 | 状态 |
|--------|------|------|------|
| **P1** | 教学日历 + 学期计划 | 备课不再是随机任务，而是有计划的学期工作 | [ ] |
| **P1** | 导出后 Word 修改回导（diff & merge） | 闭合备课→提交→修改→定稿循环 | [ ] |
| **P2** | 手机端轻量编辑 | 覆盖通勤、排队、睡前等碎片化使用场景 | [ ] |
| **P2** | 同课多班差异化 | 同一课题教多个班，自动生成差异化版本 | [ ] |

### 第三阶段：建立不可逆依赖（让老师走不了）

目标：形成竞品无法复制的结构性锁定。

| 优先级 | 功能 | 理由 | 状态 |
|--------|------|------|------|
| **P2** | 深度风格模型（few-shot 级） | 10 次备课后 AI 写的就是你的风格——AI 时代最强锁定 | [ ] |
| **P2** | 教学反思 → 下次备课优化 | 形成"使用→反馈→改进"的数据飞轮 | [ ] |
| **P2** | 全年级/全学期备课包一键生成 | 机构老师最大的效率 dream | [ ] |

## 当前 LessonPilot 的结构性优势（已具备，需持续巩固）

1. **结构化 section + pending/confirmed 工作流** — 竞品 AI 多是"一键生成整篇"，老师改起来很痛苦
2. **"文档桌"设计方向** — Google Docs 手感 + 结构化内核，竞品要么是 LMS 界面要么是课件编辑器
3. **轻量独立** — 不需要学校采购、IT 部署、培训，老师自己就能用
4. **风格记忆的长期潜力** — 大平台不会关注"这个老师的个人风格"
5. **专注语文** — 篇目级别的深度是科大讯飞做不到的

## 实施约束

- 每阶段完成后需用户手动验收，不可自动进入下一阶段
- 每个 P0/P1 功能需独立的规格文档（`docs/specs/`）
- 不破坏现有主链路：创建→生成→确认→导出
- 风格记忆（Phase 14）作为基础设施持续增强，贯穿三个阶段
