# Phase 14：老师个人风格记忆

## Summary

Phase 14 让 LessonPilot 从“能生成一份合格文档”继续前进到“越来越像老师自己的备课桌”。v1 先做老师显式维护的个人风格档案：目标写法、教学过程口吻、学校提交措辞、常用活动偏好和禁用套话。生成和局部重写时读取该档案作为 prompt 约束。

## Teacher Value

老师迁移到新工具时最担心的是：

- 生成内容像模板，不像自己的写法。
- 学校常用术语、提交口吻和栏目措辞总要手改。
- 教学目标和过程活动每次都要重新调整到自己习惯。

风格记忆的作用是减少这些重复调整，让老师产生“它懂我怎么写”的感觉。

## V1 Scope

### In

- 用户私有风格档案。
- 设置页编辑入口。
- 可启用 / 关闭。
- 支持以下字段：
  - 目标写法偏好。
  - 教学过程风格。
  - 学校提交措辞。
  - 常用课堂活动。
  - 尽量避免的套话。
- 生成和局部重写注入风格提示。

### Out

- 不做跨用户共享。
- 不从公共 RAG 或公共样例学习个人风格。
- 不做完全自动学习和不可解释画像。
- 不做学校级统一风格管理。
- 不让风格覆盖硬性质量规则、知识引用或学校模板格式。

## Product Rule

个人风格只影响表达方式和活动组织偏好，不应该降低内容质量：

- 不能让教学目标变空泛。
- 不能让教学过程少于必要环节。
- 不能覆盖 RAG 引用事实。
- 不能把 pending 内容自动确认为 confirmed。

## Acceptance Criteria

- 老师能保存、查看、修改、关闭个人风格档案。
- 生成和重写均能读取启用状态下的风格档案。
- 禁用状态下不注入风格提示。
- 风格档案严格按 `user_id` 隔离。
- 设置页说明清楚：这是生成参考，不会替老师自动确认内容。
- 后端和前端测试覆盖核心行为。
