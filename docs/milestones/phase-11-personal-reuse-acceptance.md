# Phase 11：真实样本验收与个人资料复用闭环

## Summary

本阶段把 Phase 7-10 的个人资料库从“能上传保存”推进到“能影响下一次备课”。老师上传过的旧教案、讲义或 PPT 大纲，应能在同主题备课时被推荐、参与生成、留下 section 级引用，并在质量检查后提供可执行的待确认修订。

## Key Outcomes

- 个人资料库新增当前用户私有推荐能力，基于课题、标题、栏目和正文进行可解释匹配。
- 生成链路支持参考个人资料，并通过 `asset_status` 告诉老师资料命中、未命中、禁用或降级。
- `section_references` 能区分公共知识库资料与“我的资料”。
- 质量检查 2.0 的高频问题可以一键生成待确认修订，不自动替老师验收。
- 新增脱敏合成样本和验收脚本，覆盖学校模板、旧教案、PPT 大纲、个人资料复用和导出前体检。

## Scope

Phase 11 仍聚焦语文 MVP。个人资料复用 v1 使用关键词、标题、主题和栏目正文的轻量匹配，不强依赖 embedding；个人资料默认私有，不混入公共知识包；不做真实 `.pptx` 导出、PDF/OCR、学校后台、团队共享或全学科扩张。

## Acceptance

- 上传旧 `.docx` 或 `.pptx` 后，下一次同主题备课能看到个人资料推荐。
- 开启“参考我的资料库”后，生成过程能显示个人资料命中/未命中状态。
- 至少 2 个生成 section 能留下“我的资料”引用，并在前端与公共知识库引用区分展示。
- 其他用户无法推荐、检索、引用当前用户的个人资料。
- 至少 3 类质量检查问题能一键生成待确认修订，且不自动覆盖 confirmed 状态为已确认。
- 现有学校模板、旧教案导入、上课包、默认 Word 导出、公共 RAG 引用链路不退化。
