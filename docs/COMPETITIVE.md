# 竞品迁移策略

## 目标

让 ClassIn、科大讯飞、希沃等平台的重度备课用户愿意迁移到 LessonPilot，并在使用后形成依赖。

LessonPilot 不靠“大而全”赢，而靠“更懂老师写教案、改教案、交教案”赢。

## 竞品强项

### ClassIn 类平台

强项：
- 在线课堂。
- 互动教学。
- 课程和学生进度组织。

LessonPilot 避免正面硬碰：
- 不做完整直播课堂。
- 不做班级管理系统。

迁移突破口：
- 让老师更快生成正式教案/学案。
- 把旧教案、讲义、PPT 变成可复用资产。

### 科大讯飞类平台

强项：
- 智慧课堂生态。
- 资源和数据平台。
- 学校级品牌信任。

LessonPilot 避免正面硬碰：
- 不做学校级大平台。
- 不做重数据分析管道。

迁移突破口：
- 更轻、更专注老师个人备课。
- 每一节内容可编辑、可确认、可导出。
- RAG 引用让内容来源更透明。

### 希沃类平台

强项：
- 课件制作。
- 白板工具。
- 云资源。

LessonPilot 避免正面硬碰：
- 不先做完整课件编辑器。

迁移突破口：
- 先生成高质量教案/学案。
- 输出 PPT 大纲和课件文案，承接到老师已有课件工具。
- 长期再考虑课件模板。

## 迁移门槛

老师不愿迁移的原因通常是：

- 旧 Word 教案很多。
- 学校模板已经固定。
- PPT 和讲义都在旧平台或本地。
- 害怕新工具生成内容还要大改。
- 不想重新建立资料库。

## LessonPilot 应优先消除的门槛

1. 旧 Word 教案导入。
2. 学校 Word 模板识别与保存。
3. PPT/讲义反向生成教案或学案。
4. 个人知识资料入库。
5. 生成内容质量体检。

## 不可逆爽点

老师应该逐步感觉到：

- 它记得我的学校格式。
- 它越来越像我的写法。
- 它能复用我以前的资料。
- 它导出的 Word 可以直接交。
- 它知道哪些内容参考了资料。

只要这些点成立，老师就会把 LessonPilot 当成自己的备课桌，而不是一次性生成器。
