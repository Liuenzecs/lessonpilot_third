# LessonPilot Design System — 教师文档桌

## 1. 设计定位

LessonPilot 的界面不再追求“像某个通用知识库”，而是要像老师每天打开的一张可靠备课桌：左边有目录，中间是能阅读和修改的纸张，右边是能不能交、用了哪些资料、导出格式是否正确的检查区。

核心体验是 **Google Docs 手感 + 结构化备课内核**：

- 视觉上像在线文档，减少面板堆叠和卡片套卡片。
- 信息架构上仍保留 section、pending/confirmed、引用、质量检查、学校模板、个人资料库。
- AI 是后台整理能力，不是页面主角；老师看到的是文档、证据和下一步动作。

## 2. 视觉原则

- **文档优先**：编辑器中心永远是纸张画布，内容要能像教案一样被扫读。
- **三栏清晰**：左侧目录定位，中间正文编辑，右侧检查/引用/导出/资料辅助。
- **老师下一步明确**：每个页面都回答“现在该做什么”，避免功能入口平均用力。
- **低装饰高可读**：不用大面积渐变、装饰插画、悬浮大卡片；用分隔、留白、状态色和文档层级建立秩序。
- **结构不藏起来**：待确认、已确认、引用来源、导出风险必须可见，但不打断正文阅读。

## 3. 色彩角色

- **Paper White** `#ffffff`：文档纸张、主要输入区域。
- **Desk Background** `#f4f6f8`：应用背景，弱化 chrome，让纸张更突出。
- **Panel Surface** `#f8fafc`：左侧目录、右侧检查栏、工具区。
- **Ink** `#172033`：正文和标题。
- **Muted Ink** `#64748b`：辅助说明、元信息。
- **Teaching Blue** `#2563eb`：主操作、当前定位、可点击证据。
- **Ready Green** `#15803d`：可提交、已确认、通过检查。
- **Warning Orange** `#d97706`：待确认、提醒、需修复。
- **Blocker Red** `#dc2626`：阻断导出、错误、危险操作。
- **Line** `#d8dee8`：文档和面板分隔线。

状态色只用于状态和行动，不作为大面积背景主题。

## 4. 布局规范

### 公域页

- 首屏必须出现真实产品形态：备课文档、目录、质量检查、引用、导出。
- 标题表达具体结果，例如“把旧教案整理成能提交的 Word”，不要只说 AI。
- CTA 聚焦试做教案、导入旧教案、登录继续。

### 备课台

- 定位为“备课队列”，不是卡片画廊。
- 任务卡显示：课题、年级学科、文档类型、最近更新时间、下一步动作。
- 工具入口放在顶部工具区：开始备课、导入旧教案、学校模板、个人资料库。

### 创建页

- 定位为“备课启动台”。
- 课题输入是主视觉焦点；学科、年级、课型、文档类型是紧凑控件。
- 右侧展示自动匹配信息：系统模板、个人资料、知识增强提示。

### 编辑器

- 顶部：文档标题、保存状态、主要操作。
- 左侧：文档目录和 section 状态。
- 中间：纸张画布，section 像文档段落而不是独立卡片。
- 右侧：检查栏，承载质量体检、学校模板、个人资料、上课包、引用说明。
- 模态只用于历史、导出预览等需要完整空间的任务。

## 5. 组件规范

- 按钮半径保持 6-8px，工具栏按钮可更紧凑。
- 卡片半径不超过 8px；不要在卡片里再嵌套大卡片。
- 状态标签使用小尺寸胶囊，但正文区域避免标签泛滥。
- 图标优先使用 `lucide-vue-next`，仅在没有合适图标时使用文字。
- 输入框、选择器、textarea 使用稳定高度和清晰 focus ring。
- 表格型教学过程要像文档表格，边框清晰、列标题紧凑。

## 6. 字体与排版

- 全局使用 Inter / 系统字体。
- 文档正文以 15-16px、1.65-1.8 line-height 为主。
- 页面大标题只用于公域页和页面头；编辑器内部标题应克制。
- 不使用负 letter-spacing 作为默认风格；中文文本优先保持自然字距。

## 7. 响应式

- 手机端可浏览公域页、备课台、创建页、资料库。
- 完整编辑器继续以平板/桌面为主；手机端显示明确提示和返回入口。
- 宽屏编辑器保持三栏，中等屏幕右侧栏可下沉，左侧目录可折叠。

## 8. 验收口径

一次 UI 变更只有同时满足以下条件才算完成：

- 老师能在首屏理解产品是“备课文档 + 质量检查 + Word 导出”。
- 备课台能看出每份任务的下一步动作。
- 创建页不再像长表单，课题和匹配资料成为主流程。
- 编辑器像在线文档，且 section 状态、引用、质量检查不被隐藏。
- 现有创建、生成、改写、确认、导出、旧教案导入、学校模板、个人资料库功能不退化。
